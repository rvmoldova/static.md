## Project Context

Static.md is a free, open-source image hosting service. No public API — the API is internal only. No user accounts or registration. The source code is public but the service itself is a hosted product at static.md.

## Design Context

### Users
Developers and technical users sharing screenshots, code snippets, and debug images. They value speed and precision — drag-drop or paste an image, get a link, move on. The terminal-inspired aesthetic signals "built by developers, for developers." Context is typically mid-workflow: they need to share something fast without leaving their mental model.

### Brand Personality
**Precise, quiet, confident.** Like a well-configured terminal — no noise, just function. The interface should feel like it respects the user's time and attention. Nothing flashy, nothing gratuitous. Every element earns its place.

### Aesthetic Direction
**Terminal Elegance** — the design system's internal name. Monospace typography (Menlo, self-hosted) is the entire type stack. Uppercase headings with tight tracking. oklch color space throughout. Cool blue primary (hue 230), sage green accent (hue 155), terracotta warm accent (hue 25). Never pure black or white — surfaces are tinted toward hue 250.

**References:** Linear, Vercel — clean developer tools with subtle depth, restrained animation, and confident whitespace. The color-cycling upload button is the one permitted moment of expressiveness.

**Anti-references:** Imgur, Gyazo, generic image hosts. Avoid cluttered dashboards, ad-driven layouts, or anything that feels disposable. Also avoid neon-on-dark "hacker" aesthetics — the dark theme deliberately desaturates.

**Theme:** Light and dark, toggled via `data-theme` attribute, OS preference honored. Dark mode uses surface lightness for depth hierarchy instead of box shadows.

### Design Principles

1. **Monospace is the medium.** Every piece of text uses `--font-mono`. Typography choices (weight, case, spacing) are the primary visual tools — not color splashes or decorative elements. Italic monospace is avoided (use color instead).

2. **Quiet depth, not flat.** Surfaces use subtle oklch tinting (hue 250) and layered lightness to create depth without heavy shadows. Light mode gets soft shadows; dark mode gets none — only surface lightness differentiates layers.

3. **Motion with purpose.** Only transform and opacity are animated (GPU-composited, no layout thrashing). `prefers-reduced-motion` is fully respected. Entrance animations use staggered `fade-in-up` at 80ms intervals. The upload button's color cycle is the single persistent animation — it draws the eye to the primary action.

4. **Accessibility is structural.** Push toward WCAG AAA. Focus-visible outlines on all interactive elements. Screen-reader-only utilities for decorative content. Keyboard navigation for all flows. Optical weight compensation in dark mode (`--weight-regular: 350`).

5. **Terminal metaphors, not cosplay.** Use terminal idioms where they add clarity ($ prompts, em-dash lists, `›` active indicators, `kbd` styling, blinking cursors) but don't simulate a literal terminal. The interface is a modern web app that borrows the terminal's information density and precision.

### Icon System

All icons use **Lucide** (`lucide-vue-next`). No inline SVGs. Lucide uses stroke-based rendering (`currentColor` inherited). There is no global `svg { fill }` rule — Lucide handles its own fill/stroke. Import only what you need per component.

### Micro-interactions

Delight is **subtle sophistication**, not playful:
- **Copy feedback:** Buttons swap to a checkmark (sage green accent) for 1.5s after copying
- **Gallery images:** Fade in on load with staggered delay; 3D hover depth via `perspective` + `translateZ`
- **Upload overlay:** Breathing border pulse, floating cloud icon
- **Upload progress:** Shimmer highlight on progress bar; accent green flash on completion
- **Page transitions:** `translateY(12px) scale(0.99)` entrance
- **Hero:** Blur-to-sharp reveal with stagger

All animations are GPU-composited (transform, opacity, filter only). All respect `prefers-reduced-motion`.
